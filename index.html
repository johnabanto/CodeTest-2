AngularJS Q&A

1. How do you define an AngularJS application?
Assigning the ng-app directive defines an AngularJS application.

2. What is a controller?
A controller is a JavaScript Object containing attributes, properties and functions and is defined using ng-controller directive.

3. What does ng-model do?
The ng-model directive binds the value of HTMl controls (input, select, textarea) to application data.

4. What is interpolation markup?
Interpolation markup is used by AngularJS to provide data-binding to text nodes and attributes values.

5. What is $scope?
$scope is the link between your view and controller. Any property that the HTML evaluates is evaluated on a particular scope.

6. What is dependency injection?
Dependency Injection is a software desing pattern in which components are given their dependencies instead of hard coding them within the components.

7. What is two-way binding?
Two-way binding is the synchronization between the model and the view.

8. What is 'controller as' syntax?
The 'controller as' syntax defines a named scope and you can use this with the help of ng-controler directive. It defines properties on the functions (controller), not the $scope.

9. What is MV* architecture
MV*(Whatever) or Model View Whatever is a term used to indicate the capability of the framework being used to give the option to choose from MVC or MVVM approaches. Whatever is a term used to say you can used what system works for you.

10. What is the difference between a service and a factory?
Factory can be used when you have a constructor function which needs to be instantiated in different controllers. Service is a kind of singelton object. The object return from Service will be same for all controller.
